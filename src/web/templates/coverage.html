Coverage

Transparent list of what the platform scans and what it does not. No marketing fluff — actual capabilities.

What we scan

  • CVE / advisories: NVD (last 30 days), GitHub Security Advisories, OSV, AlienVault OTX. Matched to your tech stack and scope.
  • Dark Web: Tor-routed search for your domains, company names, and tech; leak indicators. Tor killswitch: no clearnet fallback.
  • Exploit matching: GitHub repos (exploit, poc, payload) matched to your fingerprint; safe probe only (no destructive payloads).
  • Fuzzer: Context-aware payloads; LFI/path traversal, SQLi, XSS, etc. Validated by response signature before reporting.
  • Recon: Certificate Transparency (crt.sh), DNS, subdomains; S3/Azure/GCP bucket discovery; IP-to-org mapping.
  • Supply chain & secrets: NPM/PyPI typosquatting; GitHub code search for leaked secrets (API keys, .env).
  • HIBP: Breached domains in scope (when API key configured).

What we do not do

  • We do not run destructive or modifying payloads (e.g. DROP TABLE, delete file). Safe probe and signature-based validation only.
  • We do not scan all 65535 ports by default; default is 80, 443, 8080; extended port list available with deep scan.
  • We do not receive real-time webhooks from NVD/GitHub; we poll on a schedule (orchestrator / auto_check).
  • We do not include synthetic, mock, or example findings in reports; all data is from live DB and feeds.

Verified vs potential

Findings that pass signature validation (fuzzer) or PoC check are treated as Verified. Only verified findings are included in the final report. Potential or unvalidated items are not presented as confirmed.
