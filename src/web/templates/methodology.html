Methodology

How findings are discovered, correlated, and validated. This describes the actual engine behavior.

Correlation

Feeds (NVD, GitHub Advisories, OSV, AlienVault OTX, HIBP) are fetched with short-lived caching. Findings are matched to your scope using tech stack (word-boundary matching) and domains. Only findings that match your configured clients and technologies are included.

Fingerprinting

Before CVE matching, targets are fingerprinted (HTTP headers, meta tags). The detected tech stack is stored and used for correlation. If the Rust engine is unavailable, a Python fallback (Server, X-Powered-By headers) is used.

Fuzzer & validation

Fuzzer runs context-aware payloads (from payload_signatures.json). Anomalies (e.g. status 500, length change) are confirmed by payload-specific signature matching in the response body (e.g. LFI requires root:x: in body). Only confirmed anomalies are reported; no synthetic or mock findings.

Severity

Severity comes from the feed (NVD CVSS, GitHub, OSV) or from the fuzzer anomaly type. CVSS and EPSS are used where available for priority scoring.

Reports

PDF reports use only data from the database for the selected run. Benchmark sector averages are industry reference values, not your organization's data. Every report includes a SHA-256 integrity stamp.
