# SOC Command Center — 5 Advanced Cyber Engines (Backend Architecture)

**No mock data. No fake results. All engines are live-only; empty results = verified 0 findings.**

**Implementation:** All 5 engines (supply_chain, ollama_fuzz, bola_idor, osint, asm) run in **Rust** (`fingerprint_engine` binary). Python only invokes the binary and parses JSON; no scan/attack logic in Python for these engines.

---

## 1. Verified Supply Chain Auditing

**Module:** `src/engines/supply_chain_engine.py`  
**Entrypoint:** `run_supply_chain_audit(target, scope=None) -> { status, findings[], message }`

- **Goal:** Reduce false positives to 0% by verifying whether a vulnerable CVE function is *actually called* in the target's code.
- **Planned implementation:**
  - AST (Abstract Syntax Tree) parsing of application code (e.g. via `ast` or tree-sitter) to build call graphs.
  - Deep dependency graphing (transitive deps from requirements.txt / package-lock / Gemfile).
  - Map CVE-affected symbols to call sites; only report when the vulnerable symbol is reachable.
- **Existing:** `src/supply_chain.py` — NPM/PyPI/OSV footprint and typosquatting; can be integrated into this engine.
- **Limits:** None; built for deep, unconstrained recursive discovery within authorized scope.

## 2. Zero-Cost AI Fuzzing (Ollama)

**Module:** `src/engines/ollama_fuzz_engine.py`  
**Entrypoint:** `run_ollama_fuzz(target, scope=None) -> { status, findings[], message }`

- **Goal:** Context-aware fuzzing payloads (XSS, SQLi, SSRF) generated by a local AI model to bypass WAFs; no paid APIs.
- **Planned implementation:**
  - Send target HTTP response (headers + body snippet) to local Ollama REST API (`http://127.0.0.1:11434`).
  - Prompt: ask the model to generate sophisticated, context-aware payloads for the given response.
  - Execute generated payloads against target (via existing fuzzer or new runner); record anomalies.
- **Config:** `OLLAMA_BASE` (default localhost:11434); no API key.
- **Limits:** None; payload count and depth driven by model and response size.

## 3. Smart Logic Attacks (BOLA/IDOR)

**Module:** `src/engines/bola_idor_engine.py`  
**Entrypoint:** `run_bola_idor(target, scope=None) -> { status, findings[], message }`

- **Goal:** Automatically find Broken Object Level Authorization and Insecure Direct Object Reference via parameter analysis and privilege escalation attempts.
- **Planned implementation:**
  - Ingest OpenAPI/Swagger spec (from target URL or uploaded spec).
  - Parse traffic logs (or proxy logs) to identify parameter patterns: User IDs, UUIDs, session tokens, JWTs.
  - Replay requests with substituted IDs/tokens to test cross-tenant or elevated access.
  - Report when a different user’s resource is returned or a privilege boundary is crossed.
- **Data sources:** Live spec URLs and/or log files; no dummy data.
- **Limits:** None; full enumeration of identified ID-like parameters within scope.

## 4. Deep OSINT & Dark Web Crawler

**Module:** `src/engines/osint_engine.py`  
**Entrypoint:** `run_osint_crawl(target, scope=None) -> { status, findings[], message }`

- **Goal:** Certificate Transparency logs, public paste sites, and live threat intel feeds for hidden domains and leaked tokens.
- **Planned implementation:**
  - **crt.sh:** Query Certificate Transparency for subdomains/certs related to target domain.
  - **Paste sites:** Scrape public paste bins (with strict timeout and size limits) for leaked tokens, .env patterns, and target-related keywords.
  - **Threat intel:** Hook into existing or new feeds (e.g. AlienVault OTX, OSV, internal harvester).
- **Existing:** `src/recon_engine.py`, `src/darkweb_intel.py`, `src/intel_harvester.py` — integrate and extend; no dummy OSINT.
- **Limits:** No hardcoded small arrays; built to scale (pagination, streaming, configurable timeouts).

## 5. Complete Attack Surface Management (ASM)

**Module:** `src/engines/asm_engine.py`  
**Entrypoint:** `run_attack_surface_scan(target, scope=None) -> { status, findings[], message }`

- **Goal:** Full port scanning, DNS zone transfers, and technology fingerprinting across all discovered subdomains.
- **Planned implementation:**
  - **Port scanning:** python-nmap or masscan wrapper; configurable port range (e.g. top 1000 or full); concurrency limits to avoid network stack overload.
  - **DNS:** Zone transfer attempts (AXFR); subdomain enumeration from ASM + OSINT (e.g. crt.sh).
  - **Fingerprinting:** Reuse existing Rust/Python fingerprinting (Server, X-Powered-By, meta tags) across all discovered hosts.
- **Existing:** `src/recon_engine.py`, `src/fingerprint.py`, Rust `fingerprint_engine` — orchestrate from this engine.
- **Limits:** No small hardcoded arrays; deep, unconstrained recursive discovery within authorized scope.

## API & Task Flow

- **POST /api/command-center/scan**  
  Body: `{ "engine": "<id>", "target": "<url or scope>" }`.  
  Validates `engine` ∈ { supply_chain, ollama_fuzz, bola_idor, osint, asm }.  
  Dispatches **Celery** task `weissman.run_soc_engine` (queue: `orchestrator`).  
  Returns **202** with `{ job_id, engine, status: "queued" }` or **503** if queue unavailable.

- **Celery task:** `weissman.run_soc_engine(engine_id, target, scope=None)`  
  Calls the corresponding engine entrypoint; returns `{ status, findings[], message }`.  
  Exceptions are caught and returned as `status: "error"` with `message`; no mock fallback.
